Загрузка файлов импорта

Соглашение о поддерживаемых форматах

  1. Загружаемый файл не должен превышать 5 Мбайт
  2. Загружаемый файл должен быть представлен в следующих форматах CSV или TXT
  3. Число строк в файле не должно превышать 10 000

Соглашение о станадартах содержимого в файлах формата TXT

  1. На каждой строке должно быть указано одно URL с учетом ведущего слеша

Соглашение о станадартах содержимого в файлах формата CSV

  1. В качестве разделителя используется символ ; (точка с запятой)
  2. Каждая строка должна быть представлена в виде набора из 5 параметров:
    1. ID документа (любое целое число)
    2. Флаг сохранения GET параметров (1 - сохранять, 0 - не сохранять)
    3. Флаг поиска правила с учетом GET параметров (1 - учитывать, 0 - не учитывать)
    4. Фалаг активности правила (1 - активно, 0 - не активно)
    5. URL (любая строка)

Обратите внимание, что если вы добавляете URL с GET параметрами, то флаг поиска правил с учетом GET параметров должен быть выставлен в значение 1. Для примера, вы добавили ссылку /index.php?post=1&categoryID=22 которая будет вести на документ ID 3 со ссылкой /shop/phone.html. При этом флаг поиска правила с учетом GET параметров выставлен в значении 0. Таким образом, при заходе на сайт по ссылке /index.php?post=1&categoryID=22, будет произведен поиск правила по URI /index.php. И именно для того, чтобы поисходил поиск правила по всей ссылке, нужно выставить флаг в значение 1.

Если же выставить флаг сохранения GET параметров в значение 1, то на примере предыдущей предыдущих ссылок, произойдет редирект с адреса /index.php?post=1&categoryID=22 на /shop/phone.html?post=1&categoryID=22 Это может быть полезно на страницах с фильтрами (когда адрес страницы изменился, а фильтры остались прежними), но в большинстве случаев это лишнее и поэтому рекомендуется выставлять флаг сохранения GET параметров в значение 0.

Внимание! После выбора файла он будет автоматически загружен и обработан.
